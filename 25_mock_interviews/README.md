# Phase 25: Mock Interviews

## Topics
- Timed practice (45 min per problem set)
- Communication during problem solving
- Clarifying questions to ask
- Complexity discussion
- Trade-off analysis

## Weekly Mock Interview Format

### Structure (60 minutes)
| Time     | Activity                                    |
|----------|---------------------------------------------|
| 5 min    | Read problem & ask clarifying questions     |
| 5 min    | Discuss approach with interviewer           |
| 25 min   | Code the solution                           |
| 10 min   | Test with examples & edge cases             |
| 10 min   | Discuss complexity & optimization           |
| 5 min    | Follow-up question                          |

## Tips for Mock Interviews
- Always think out loud
- Start with brute force, then optimize
- Discuss time & space before coding
- Handle edge cases explicitly
- Practice writing clean, readable code
- Never stay silent for more than 30 seconds
