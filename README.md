# DSA Interview Roadmap (Data Engineer & AI/ML Focus)

**Goal:** Master interview-oriented DSA in ~1 hour/day through concepts, patterns, and curated practice rather than memorizing hundreds of problems.

1. Programming Logic & Patterns
2. Time & Space Complexity
3. Arrays
4. Strings
5. Recursion
6. Searching
7. Sorting

10. Sliding Window
11. Linked List
12. Stack
13. Queue & Deque

15. Trees
16. Binary Search Trees
17. Heap / Priority Queue
18. Trie
19. Graph
20. Greedy

## Daily 1-Hour Study Structure

| Time   | Activity                              |
|--------|---------------------------------------|
| 10 min | Learn one concept                     |
| 15 min | Understand intuition & dry run        |
| 25 min | Solve 2-3 interview questions         |
| 10 min | Revision, notes & complexity analysis |

## Learning Template (Use for Every Topic)

1. Problem Statement
2. Intuition
3. Visualization
4. Dry Run
5. Algorithm
6. Python Implementation
7. Time & Space Complexity
8. Edge Cases
9. Interview Variations
10. Real-world Use Cases

## Roadmap Philosophy

Focus on understanding patterns instead of memorizing solutions. Each topic should be learned through intuition, visualization, dry runs, implementation, complexity analysis, and interview-focused variations. Aim to master ~250–300 carefully selected questions rather than solving thousands mechanically.
